## 源码阅读

### FPU

内核中一般禁止使用浮点寄存器或指令（包括 C 语言的 float 和 double 数据类型），这是为了减少系统调用开销，内核无需保存和恢复用户空间的浮点寄存器。

当然如果驱动或内核程序需要包含浮点代码，那需要将包含浮点代码的函数隔离单独的源文件（编译时需要另外的编译参数），同时在调用这些函数时使用浮点的“临界区”。

具体可以参考[FP API 内核文档](https://www.kernel.org/doc/html/next/core-api/floating-point.html)

简单总结：
- 内核一般无浮点，这仅为了方便系统调用，对于上下文的切换仍有浮点切换。
- 如有浮点需要，必须使用相关函数保证临界区，并直接在编译阶段就指定编译浮点参数。
- 因此以具体架构为例：
  1. risc-v 的内核中一般在源文件编译阶段时不需要指定浮点，而在汇编阶段需要加上浮点支持（旧内核）；最新内核中一般C源文件不指定浮点参数，汇编文件以及需要浮点的C源文件会指定浮点参数（一定开启 `CONFIG_FPU`，否则用户态和内核态都不能使用浮点，FPU寄存器会随切换而直接切换）。
  2. arm(32) 的最新内核中默认参数可能已经使用了 vfp，但仍然遵守文档中的规定，对于要浮点的文件需要按规定处理（一定开启 `KERNEL_MODE_NEON`，内核态才能使用浮点，用户态只要有使能浮点单元就可以用浮点计算，使用 lazy context switching（懒切换）方式，在切换中使用了通知链的方式，通知链为 `thread_notify_head`）。
  3. arm64 遵守文档中的规定，对于要浮点的文件需要按规定处理（一定开启 `KERNEL_MODE_NEON`，内核态才能使用浮点，用户态只要有使能浮点单元就可以用浮点计算，但也是使用 lazy context switching（懒切换）方式，具体是在切换中直接调用函数 `fpsimd_thread_switch`）。
